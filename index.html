What is OpenStack?

  • An IaaS platform to build clouds
  • Meant to be used for all types of clouds (public/private/hybrid)
  • Simple to implement (getting there!) and operate
  • Feature rich
  • Massively scalable

What is OpenStack? (cont)

  • Base building blocks are compute, networking and storage
  • OpenStack services exposed via APIs
  • Users and applications interact with OpenStack via dashboard, CLIs or various available SDKs

What is OpenStack? (cont)

Services available on OpenStack clouds thru APIs:

  • Nova (Compute as a service)
  • Neutron (Network as a service)
  • Cinder (Block-storage as a service)
  • Keystone (Identity service)
  • Glance (Image service)
  • Swift (Object storage)
  • Heat (Orchestration as a service)
  • Trove (Database as a service)
  • Sahara (Big data processing as a service)
  • Magnum (Container orchestrator engine as a service)
  • Many many more...

What is OpenStack? (cont)

What users and companies do with OpenStack

  • Automate, automate and automate datacenter operations and maximize resources usage
  • Foundation for virtualizing network functions
  • Provide self-service resources to developers to accelerate product development
  • Provide public cloud services
  • Package and curate OpenStack in distros to provide private cloud services
  • Discover the secrets of the universe (for real, check next slide!)

Common misconceptions about OpenStack

  • I can create VMs with ESX/KVM/$VIRT, why use OpenStack?
  • AWS is awesome and super easy to use, why bother installing OpenStack?
  • Docker/Mesos/Kubernetes/containers is da future, OpenStack is dead
  • OpenStack is too difficult to install, need experts to run it

Why use OpenStack?

  • Open source
  • Open design
  • Open development
  • Open community
  • And most importantly: OpenStack != Code, OpenStack == People

Workshop 2

Install DevStack and explore

      ssh test@<your public instance IP>
      sudo apt-get update && sudo apt-get -y install git
      git clone git://git.openstack.org/openstack-dev/devstack
      cd devstack
      vi local.conf
      (Put local.conf contents from https://wiki.openstack.org/wiki/NeutronDevstack)
      ./stack.sh
      screen -x

  • Check DevStack docs for more information: http://docs.openstack.org/developer/devstack/
  • Check how to source the DevStack clouds credentials
  • Check the various screen windows
  • Stop and start a foreground service

Workshop 2

Keystone

      openstack --os-cloud devstack-admin endpoint list
      openstack --os-cloud devstack-admin endpoint get <endpoint uuid>
      openstack --os-cloud devstack-admin domain list
      openstack --os-cloud devstack-admin domain create mydomain
      openstack --os-cloud devstack-admin project list
      openstack --os-cloud devstack-admin project create myproject --domain mydomain
      openstack --os-cloud devstack-admin user list
      openstack --os-cloud devstack-admin user create myuser --domain mydomain --project myproject --password secret
      openstack --os-cloud devstack-admin user show myuser

Workshop 5

Nova

      openstack --os-cloud devstack-admin server list
      openstack --os-cloud devstack-admin keypair list
      openstack --os-cloud devstack-admin flavor list
      ssh-keygen -t rsa -N ""
      openstack --os-cloud devstack-admin keypair create mykey --public-key ~/.ssh/id_rsa.pub
      openstack --os-cloud devstack-admin server create myserver --flavor d1 --key-name mykey --image ubuntu-trusty --nic net-id=private
      openstack --os-cloud devstack-admin ip floating create public
      openstack --os-cloud devstack-admin ip floating list
      openstack --os-cloud devstack-admin ip floating add <floating ip from previous command> myserver
      openstack --os-cloud devstack-admin security group create ssh
      openstack --os-cloud devstack-admin security group rule create --src-ip 0.0.0.0/0 --dst-port 22 --ingress ssh
      openstack --os-cloud devstack-admin server add security group myserver ssh
      ssh -i ~/.ssh/id_rsa ubuntu@<floating ip created earlier>
